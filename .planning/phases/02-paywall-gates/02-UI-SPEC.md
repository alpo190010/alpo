## Spacing Scale

Declared values (multiples of 4 only). Pre-populated from existing component patterns in `IssueCard.tsx` and `CTACard.tsx`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label padding |
| sm | 8px | Signal row vertical padding (`py-1.5`), tight stacking |
| md | 16px | Default element spacing, card interior stacking |
| lg | 24px | Card internal padding (`p-6`), section-level vertical rhythm |
| xl | 32px | Section padding, grid gap base |
| 2xl | 48px | Major section breaks, page-level vertical rhythm |
| 3xl | 64px | Page-level bottom padding |

Exceptions:
- Compact card variant uses `p-5` (20px) on smaller breakpoints — this is a one-off exception to the lg token, not a new token. Executor must not introduce a separate token for this value.
- Locked card minimum touch target: 44px height (WCAG 2.5.5 AAA — clickable entire card surface, not just icon)
- IssueCard animation stagger: `index * 80ms` delay (existing pattern, not a spacing token)
- Card border radius: `rounded-2xl` (16px) — consistent across IssueCard, CTACard, and all new locked-state elements

## Typography

Pre-populated from verified usage in `IssueCard.tsx`, `CTACard.tsx`, and `globals.css`.

Two weights only: 400 (regular) and 800 (extrabold). Caption labels are distinguished from body by uppercase + letter-spacing, not a third weight. The 700 bold weight is removed — all heading, score, and label roles use 800.

| Role | Size | Weight | Line Height | Font Family | Usage |
|------|------|--------|-------------|-------------|-------|
| Caption/label | 14px (sm) | 800 (extrabold) | 1.0 (uppercase, tracked) | Inter | Score label, impact label, "Est. Conversion Loss" eyebrow text — uppercase + letter-spacing distinguishes from body |
| Body | 14px (sm) | 400 (regular) | 1.625 (relaxed) | Inter | Problem description text, signal row labels, locked card sub-label |
| Card heading | 16px (base) | 800 (extrabold) | 1.25 (snug) | Manrope | Dimension name in IssueCard, CTACard heading |
| Display/Score | 20px (xl) | 800 (extrabold) | 1.0 (numeric) | Manrope | Score numbers (catScore, revenue figures), CTA section heading |
| Section heading | 24px (2xl) | 800 (extrabold) | 1.2 | Manrope | "Issues Found" section heading |

Final 4-size scale: 14px (caption + body), 16px (card heading), 20px (display/score), 24px (section heading).

Locked card "Sign up to see fixes" label: 14px, weight 400, Inter, `var(--on-surface-variant)`.

## Color

Pre-populated from `globals.css` design tokens. 60/30/10 split confirmed from existing surface usage.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--background` / `--surface` | `#ffffff` | Page background, card base (`--surface-container-lowest`) |
| Secondary (30%) | `--surface-container-low` / `--surface-container` | `#f4f4f5` / `#e4e4e7` | Card hover state, icon background, border fill |
| Accent (10%) | `--primary` / `--brand` | `#27272a` (zinc-800) | Locked card hover state, focus ring, card expand border, CTACard dark background gradient |
| Destructive | `--error-base` | `#ba1a1a` | Destructive actions only (not used in this phase — no destructive actions) |

Accent (`--brand` / zinc-800) is reserved ONLY for:
- Hover border on IssueCard (`hover:border-[var(--brand)]/40`)
- Focus ring on interactive elements (`outline: 2px solid var(--brand)`)
- Icon hover color transition (`group-hover:text-[var(--brand)]`)
- Lock icon hover on locked cards (`group-hover:text-[var(--brand)]`)
- CTACard dark gradient endpoint

Semantic colors for locked card state:
- Lock icon default: `var(--on-surface-variant)` (#454556) — muted, not alarming
- Lock icon hover: `var(--brand)` — signals interactivity
- "Sign up to see fixes" text: `var(--on-surface-variant)` — subordinate to dimension name
- Impact badge HIGH: `var(--error-text)` (#ba1a1a) — visible through lock, creates urgency
- Impact badge MED: `var(--warning-text)` (#b45309)
- Impact badge LOW: `var(--success-text)` (#15803d)

## Primary Focal Point

The ScoreRing is the primary visual anchor for anonymous users on the results page. It is fully visible (D-04), renders first in the reveal sequence, and communicates the headline score before any issue card is processed. For anonymous users who have not yet converted, the revenue loss figure inside PluginCTACard serves as the secondary anchor — it makes the abstract score concrete. All other elements (locked IssueCards, CTACard) support conversion from these two anchors.

## Component Inventory

### Modified: IssueCard (locked state redesign — D-05)

**Current behavior:** Locked card renders full card body (name + problem text + revenue) with a `LockKeyIcon` replacing the bottom arrow.

**Required behavior:** Locked card collapses to a minimal 3-element layout. Full body content hidden.

Locked card anatomy (top to bottom):
1. **Row 1 — Icon + Score:** Dimension category icon (44×44px, `bg-[var(--surface-container-high)]`, `rounded-2xl`) on left; score number (20px extrabold Manrope, impact color) on right with "Score" eyebrow (14px extrabold uppercase Inter, tracked).
2. **Row 2 — Dimension name:** Category name only (16px extrabold Manrope, `var(--on-surface)`, `leading-snug`). NO problem description text beneath it.
3. **Row 3 — Bottom bar:** Impact badge on left (HIGH/MED/LOW, appropriate semantic color); `LockKeyIcon` (20×20px, `var(--on-surface-variant)`, hover: `var(--brand)`) on right.
4. **Row 4 — Lock label:** "Sign up to see fixes" (14px, 400 weight, `var(--on-surface-variant)`, centered or left-aligned flush below the bottom bar). Appears below the divider line.

Outer wrapper: identical to unlocked card — `contain-card group text-left bg-[var(--surface)] rounded-2xl p-5 sm:p-6 flex flex-col border border-[var(--outline-variant)]/20 hover:border-[var(--brand)]/40 transition-all duration-300 hover:-translate-y-1 hover:shadow-[var(--shadow-card-hover)]`. Grid layout must not shift.

Interaction: entire card is a button (`onClick` → `setAuthModalOpen(true)`). `cursor-pointer`.

Animation: `fade-in-up 400ms var(--ease-out-quart) {index * 80}ms both` — identical to unlocked cards.

### Modified: CTACard (anonymous inline CTA — D-08, D-09)

**Required behavior:** Rendered as the last item in the 3-column issues grid for anonymous users. Replaces/adapts existing CTACard component. Feels native to the layout.

Copy changes:
- Heading: "Your page has {N} issues." (dynamic, where N = `leaks.length`)
- Sub-copy: "Sign up free to see how to fix them."
- Button label: "Create Free Account"
- Button icon: `CaretRightIcon` (existing pattern)

Visual: reuse existing dark gradient card style (`linear-gradient(135deg, var(--on-surface) 0%, var(--primary-dim) 100%)`). Same grid pattern overlay, same `rounded-2xl`, same `min-h-[240px]`. No changes to visual treatment — only copy changes.

Interaction: `onClick` → `setAuthModalOpen(true)` in `signup` mode.

### Modified: AuthModal (initial mode for anonymous entry points — Open Question from RESEARCH.md)

**Required behavior:** When opened from a locked card click or CTACard click, AuthModal opens in `signup` mode, not `signin` (default).

Implementation: Add optional `initialMode?: "signin" | "signup"` prop to `AuthModal`. Default remains `"signin"` to avoid breaking existing usage. Locked cards and CTACard pass `initialMode="signup"`.

### Deleted: PaywallModal (D-12)

No visual contract needed — file deleted, no replacement UI.

### Unchanged: ScoreRing, PluginCTACard

These are fully visible for anonymous users (D-03, D-04). No changes to visual presentation or interaction.

### New interaction: "Scan Another" for anonymous users (Open Question from RESEARCH.md)

After removing the bottom gradient banner (D-10), anonymous users need a non-dead-end path if they want to try another URL before signing up.

**Required behavior:** A minimal "Analyze Another Page" button below the issues grid for anonymous users. Uses existing `Button` variant="gradient" pattern, size="lg", shape="card", className="polish-hover-lift".

Copy: "Analyze Another Page"

Placement: Centered below the issues grid section. No enclosing banner or gradient box — bare button only.

## Accessibility Contract

- All locked IssueCards: `role="button"` (via Button component), full keyboard activation (`Enter`/`Space`)
- Locked card aria: `aria-label="{dimension name} — locked. Sign up to see fixes."` (screen reader announces what's behind the lock)
- Lock icon: `aria-hidden="true"` (decorative — the card label carries the semantic meaning)
- CTACard: `type="button"`, keyboard activatable
- AuthModal: focus trap and scroll lock via Radix Dialog primitive (already implemented)
- Touch targets: minimum 44px height on all locked cards and CTACard (WCAG 2.5.5)
- Animation: all `fade-in-up` animations respect `prefers-reduced-motion: reduce` (already handled in globals.css)
- Color contrast: `var(--on-surface-variant)` (#454556) on `var(--surface)` (#ffffff) = 7.2:1 ratio — AAA
