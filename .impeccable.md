# Design Context — alpo.ai

> **Status:** Active, still evolving. Treat as current guidance, not gospel.

## Users
Shopify store owners, e-commerce sellers, DTC brand operators. Age 25–45, mobile-first, scan-not-read. Busy merchants who care about revenue, not design theory. Not developers or designers — they want to sell more product. They arrive skeptical ("another tool"), leave convinced ("this one actually knows my page").

**Context of use:** Mid-workday, between Shopify admin tabs. Often on a phone or laptop. Short attention span, high intent. They paste a URL and want answers in seconds, not minutes.

**Job to be done:** "Show me what's costing me money and tell me exactly how to fix it."

## Brand Personality
- **Confident** — trusted advisor who's seen 10,000 product pages, not a salesperson
- **Sharp** — cuts through noise, says what matters, never vague
- **Warm** — approachable, not clinical. This tool is on your side.
- **Revenue-denominated** — every insight ties back to money ("$2,800/mo" not "could be better")

**Emotional arc:** Trust → Surprise → Urgency → Action. The user should feel "this tool gets me" first, then "I need to fix this NOW."

## Aesthetic Direction

### Visual Tone
Consumer-warm, violet-forward, generous. Glass surfaces over warm cream backgrounds. Big rounded corners, confident whitespace, playful-but-purposeful motion. Think: a premium consumer tool that happens to show data, not a dashboard that happens to look nice.

### References
- **Arc Browser** — warm, opinionated, personality-forward design. Primary inspiration.
- **Notion** — approachable sophistication. Makes complex things feel simple.
- **Stripe** — information hierarchy, confident use of space (but warmer than Stripe)
- **Linear** — motion polish, speed as a design value

### Anti-References
- **Generic SaaS template** — no floating blobs, no "Powered by AI" badges, no hero illustrations of people high-fiving. The 2020–2024 startup aesthetic is banned.
- **Cheap SEO tool** — no dashboard overload, no anxiety-inducing red warnings everywhere, no 47 metrics crammed into one screen.
- **Enterprise software** — no gray-on-gray, no sidebar navigation, no "Contact Sales" energy.
- **Bloomberg terminal** — we're consumer-warm, not financial-cold. Data matters but it should feel inviting.

### Color System
Built as "Luminous Liquid" — a violet-primary palette with warm neutral surfaces.

- **Primary:** Violet `#7c3aed` (dimmed: `#6d28d9`)
- **Secondary:** Same violet range — monochromatic brand unity
- **Tertiary accent:** Deep magenta `#82002e` — used sparingly for emphasis
- **Background:** Warm cream `#fcf9f8` — NOT clinical white
- **Surface cards:** Pure white `#ffffff` with subtle ambient shadows
- **Surface containers:** Warm gray ladder (`#f6f3f2` → `#f0edec` → `#ebe7e7` → `#e5e2e1`)
- **Text:** Primary `#1c1b1b`, Secondary `#454556`, Tertiary `#767588`
- **Borders:** Subtle `#c6c5d9` — structure without weight
- **Error/Loss:** Red `#ba1a1a` (revenue loss, critical)
- **Warning:** Amber `#d97706`
- **Success:** Green `#16a34a`
- **Gradients:** Violet → violet-dim (`135deg, #7c3aed, #6d28d9`) for CTAs and hero elements

CSS custom properties are defined in `src/app/globals.css`. JS/TS tokens live in `src/tokens/design-tokens.ts`. The design tokens file is the canonical source; CSS vars should align.

> **Note:** `design-tokens.ts` still references an older "Luminous Liquid" palette (violet `#7000ff` + magenta `#b60055`). The CSS in `globals.css` has evolved to a purer violet system (`#7c3aed`). When in doubt, follow `globals.css` — it's what ships.

### Typography
- **Headlines:** Manrope (700, 800) — geometric, confident, tight tracking (`-0.02em` to `-0.01em`). CSS var: `--font-manrope`.
- **Body:** Inter — clean, professional, excellent readability. CSS var: `--font-inter`.
- **Data/Scores:** JetBrains Mono — monospace numbers for precision. CSS var: `--font-mono`.
- **Hierarchy:** Size + weight. Headlines bold/extrabold Manrope, body regular Inter. No color tricks for hierarchy.

### Spacing & Layout
- Cards with `rounded-3xl` (1.5rem) corners, glass blur effects, subtle ambient shadows
- Generous whitespace between sections, tight spacing within content blocks
- Max content width: `max-w-7xl` for full layouts, `max-w-2xl` for forms/inputs
- Responsive: 390px mobile to 1440px+ desktop. Mobile-first, never an afterthought.
- Grid system: 12-column with `md:col-span-*` breakpoints

### Motion & Animation
- CSS animations only — no JS animation libraries. Keep the bundle light.
- Staggered reveals: `fade-in-up` with incremental `animationDelay` (80ms–160ms steps)
- Phase transitions: `anim-phase-enter` / `anim-phase-exit` for section-level state changes
- Modal/drawer: custom enter/exit keyframes with `--ease-out-quart`
- Score reveal: ring progress via `stroke-dashoffset` transition (1.2s)
- `prefers-reduced-motion` always respected (global reset in globals.css)
- Timing: 200–400ms for UI transitions, 500–600ms for content reveals, 1200ms for the score moment
- No bounces, no wobbles, no particle effects

### Signature Patterns
- `.glass-card` — frosted glass with backdrop blur, violet border on hover
- `.primary-gradient` — violet gradient for CTAs
- `.liquid-btn` — gradient button with scale + shadow hover
- `.polish-hover-lift` — subtle translateY(-2px) + shadow on hover
- `.polish-focus-ring` — violet focus-visible ring with offset
- Score ring SVG with animated `stroke-dashoffset`

## What Makes It Memorable
The **score reveal moment** — ring fills, number counts up in bold type, revenue loss hits in red. The diagnostic loader that narrates what it's checking. The leak cards that are so specific the user thinks "how does it know my page?"

## Design Principles

1. **Revenue is the language** — Every metric ties back to dollars lost or gained. Abstract scores are meaningless without financial context.

2. **Specificity builds trust** — Generic advice is worthless. Specific advice ("Your hero image is 3.2MB, costing ~$280/mo") is worth paying for.

3. **Progressive disclosure** — Score first (1 number), then revenue impact (emotional hook), then dimensions (the breakdown), then fixes (the path forward). Never dump everything at once.

4. **Warm precision** — Data-dense but never cold. Every number has context. Every warning has a solution. The interface feels like a knowledgeable friend, not a robot auditor.

5. **Earn the upsell** — The free scan must be genuinely useful on its own. The paid report is for people who want *more*, not people who got *nothing*.

## Quality Bar
Flagship — this is the entire product. Every pixel matters. No placeholder states. No "good enough" copy. No unstyled error messages. If a user screenshots this and shares it, it should look intentional.

## Technical Constraints
- Next.js 16 App Router + React 19
- Tailwind CSS 4
- No heavy JS animation libraries — CSS animations + `requestAnimationFrame` only
- Performance: fast initial load, no layout shift, instant input response
- Accessibility: WCAG AA minimum, focus-visible rings, screen reader labels, reduced motion support
- PostHog for analytics (already integrated)
- Resend for email delivery
